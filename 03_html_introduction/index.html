🔰 HTML Introduction (în limba română)

În această lecție parcurgem conceptele fundamentale HTML, cu exerciții, soluții ascunse/afișabile, întrebări de interviu și un mini‑proiect cu cel puțin 10 task‑uri. Structura și stilurile sunt aliniate cu template‑ul dat.

🧠 Teorie & Exerciții (cu soluții ascunse)

1. 📄 Structura de bază a unui document HTML

Teorie: Un fișier HTML începe cu <!DOCTYPE html>, urmat de <html> (cu atribut lang), <head> (metadate) și <body> (conținut).

💡 Păstrează <meta charset="UTF-8"> și <meta name="viewport"> pentru suport limbi și responsive.

Exercițiu: Creează un document minim valid HTML5 cu titlul „Salut, HTML!”.

<!DOCTYPE html>
<html lang="ro">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Salut, HTML!</title>
</head>
<body>
  <h1>Salut, HTML!</h1>
</body>
</html>

2. 🧠 <head>, meta taguri & SEO de bază

Teorie: În <head> definim titlul, descrierea, favicon, resurse CSS/JS. <meta name="description"> ajută la SEO; <link rel="icon"> setează favicon.

Exercițiu: Adaugă o descriere și un favicon la pagina ta.

<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Portofoliul meu</title>
  <meta name="description" content="Portofoliu web cu proiecte și contact.">
  <link rel="icon" href="/favicon.ico">
  <link rel="stylesheet" href="style.css">
</head>

3. 📝 Heading‑uri & paragrafe (h1..h6, p)

Teorie: Folosește un singur h1 pe pagină pentru titlul principal; p pentru paragrafe. Menține ierarhia corectă.

Exercițiu: Scrie o secțiune cu un h2 și două paragrafe.

<section>
  <h2>Despre proiect</h2>
  <p>Acesta este un proiect demo.</p>
  <p>Îl folosim pentru a învăța HTML de bază.</p>
</section>

4. 🔗 Link‑uri & 🖼️ Imagini

Teorie: <a> creează linkuri; href definește destinația. <img> afișează imagini; alt descrie conținutul pentru accesibilitate.

Exercițiu: Adaugă un link către MDN și o imagine cu atribut alt descriptiv.

<a href="https://developer.mozilla.org" target="_blank">MDN Web Docs</a>
<img src="poze/logo.png" alt="Sigla proiectului în albastru">

5. 🧾 Liste (ul, ol, li)

Teorie: ul = listă neordonată, ol = listă ordonată. Elementele sunt li.

Exercițiu: Creează o listă cu 3 abilități și un top 3 resurse.

<ul>
  <li>HTML</li>
  <li>CSS</li>
  <li>JavaScript</li>
</ul>

<ol>
  <li>MDN</li>
  <li>FreeCodeCamp</li>
  <li>Web Dev Simplified</li>
</ol>

6. 📊 Tabele (table, thead/tbody)

Teorie: Tabelele sunt pentru date tabulare. Folosește <th> pentru headere și scope pentru accesibilitate.

Exercițiu: Creează un tabel simplu cu 2 coloane (Lecție, Status).

<table>
  <thead>
    <tr>
      <th scope="col">Lecție</th>
      <th scope="col">Status</th>
    </tr>
  </thead>
  <tbody>
    <tr><td>HTML Intro</td><td>✔️ Finalizat</td></tr>
    <tr><td>CSS Basics</td><td>⏳ În curs</td></tr>
  </tbody>
</table>

7. 🧾 Formulare de bază

Teorie: <form> grupează inputuri. <label for> se leagă de id. Atribute utile: required, placeholder, type, name.

Exercițiu: Creează un formular de contact cu nume, email și mesaj.

<form action="#" method="post">
  <label for="nume">Nume</label>
  <input id="nume" name="nume" type="text" required>

  <label for="email">Email</label>
  <input id="email" name="email" type="email" required>

  <label for="msg">Mesaj</label>
  <textarea id="msg" name="mesaj" rows="4"></textarea>

  <button type="submit">Trimite</button>
</form>

8. 🏷️ Semantic HTML

Teorie: Folosește elemente semantice (header, nav, main, section, article, aside, footer) pentru structură logică și accesibilitate mai bună.

Exercițiu: Reorganizează o pagină folosind header, nav și main.

<header>...logo + meniu...</header>
<nav>...linkuri...</nav>
<main>
  <section>Conținut principal</section>
</main>
<footer>© 2025</footer>

9. 🎬 Media: video & audio

Teorie: <video> și <audio> acceptă atribute precum controls, autoplay, loop. Include surse alternative.

Exercițiu: Inserează un video MP4 cu controale.

<video controls width="480">
  <source src="media/demo.mp4" type="video/mp4">
  Browserul tău nu suportă elementul video.
</video>

10. 🔤 Entități HTML & diacritice

Teorie: Pentru caractere speciale folosește entități: &copy;, &nbsp;, &lt;, &gt;, etc. Diacriticele sunt afișate corect cu UTF‑8.

Exercițiu: Afișează simbolul < și > într‑un paragraf.

<p>Afișăm &lt;div&gt; și &lt;/div&gt; în text.</p>

11. 🧩 Atribute, id vs class, data-*

Teorie: id e unic, class se repetă. data-* păstrează metadate personalizate.

Exercițiu: Creează un buton cu data-level="incepator".

<button class="btn" id="cta" data-level="incepator">Începe acum</button>

12. ⚙️ Încărcarea scripturilor (defer / async)

Teorie: defer amână execuția până după parsarea HTML; async rulează când fișierul e gata, ordinea nu e garantată.

Exercițiu: Leagă un fișier app.js cu defer.

<script src="app.js" defer></script>

💬 Întrebări de interviu (HTML Intro)

1. Ce face declarația <!DOCTYPE html>?

Indică browserului că documentul folosește standardul HTML5.

2. Care e diferența dintre elemente block‑level și inline?

Block ocupă lățimea disponibilă și începe pe rând nou; inline ocupă doar cât conținutul și nu începe rând nou.

3. Ce rol au atributele alt pe imagini?

Descriu imaginea pentru cititoare de ecran și când fișierul nu se încarcă; util și pentru SEO.

4. Diferența dintre id și class?

id este unic pe pagină; class poate fi reutilizată pe mai multe elemente.

5. Ce fac atributele defer și async la scripturi?

defer execută după parsarea HTML în ordinea tagurilor; async execută imediat ce se încarcă, fără garanția ordinii.

6. Ce este accesibilitatea (a11y) în HTML?

Practicile care fac conținutul utilizabil pentru toți, incluzând persoane cu dizabilități (ex: etichete, structură semantică, contrast, focus).

7. Când folosești tabele vs CSS pentru layout?

Tabele doar pentru date tabulare; niciodată pentru layout. Layoutul se face cu CSS.

8. Ce este atributul lang pe <html> și de ce contează?

Indică limba documentului (ex: ro), ajutând cititoarele de ecran și motoarele de căutare.

9. Cum legi corect un <label> de un <input>?

Setezi for="id-ul-inputului" pe label și id identic pe input.

10. Ce este <meta name="viewport"> și de ce e important pe mobil?

Controlează scalarea pe dispozitive mobile; width=device-width, initial-scale=1.0 asigură layout corect.

11. Ce diferență este între title și meta description?

title apare în tab și în rezultatele de căutare; description e rezumat folosit de motoarele de căutare.

12. Când folosești <strong> vs <b>?

<strong> are semnificație semantică (accentuează); <b> doar stil vizual fără semnificație.

13. Ce sunt atributele data-*?

Metadate personalizate atașate elementelor, accesibile în JS via dataset.

14. Cum inserezi corect un fișier CSS extern?

În <head> cu <link rel="stylesheet" href="style.css">.

15. Ce este ARIA (la nivel de bază)?

Atribute pentru a oferi informații suplimentare cititoarelor de ecran (ex: role, aria-label).

🧪 Mini‑proiect: „Pagina CV” (10+ taskuri)

Obiectiv: creează o pagină CV single‑page folosind conceptele HTML învățate.

  1. 📄 Adaugă structura minimă HTML5 (doctype, lang="ro", meta viewport, titlu).
  2. 🧭 Creează <header> cu nume + meniu (ancore către secțiuni: „Despre”, „Experiență”, „Educație”, „Proiecte”, „Contact”).
  3. 📝 Secțiunea „Despre” cu h2 + 1‑2 paragrafe și o imagine cu alt bun.
  4. 🏷️ Folosește elemente semantice (main, section, article, footer).
  5. 📋 Creează o listă neordonată cu abilități (HTML, CSS, JS).
  6. 📊 Adaugă o secțiune cu un table (ex: tabel experiență: rol, companie, ani).
  7. ✉️ Creează un formular „Contact” (nume, email, mesaj) cu label corelate la input.
  8. 🎬 Inserează un clip video de prezentare (opțional) cu <video controls>.
  9. 🔗 Adaugă linkuri către profilurile tale (GitHub, LinkedIn) cu atribute target="_blank" + rel="noopener".
  10. 🧩 Folosește data-* pe un buton (ex: data-theme) și citește‑l în JS (bonus).
  11. 🧪 Validează HTML cu W3C Validator și corectează eventualele erori.
  12. 🐙 Versiunează cu Git: 
    git init
git add .
git commit -m "feat: initialize CV page"
git branch -M main
git remote add origin <repo-url>
git push -u origin main
Hint CSS: Refolosește clasele din acest template pentru carduri/sectiuni.
Bonus JS: Adaugă un mic script care schimbă tema sau ascunde/arată secțiunea „Proiecte” folosind un buton.
