📚 Teorie — CSS Grid (în română)

CSS Grid este un sistem bidimensional pentru layout în care poți controla rânduri și coloane. Ideal pentru layout-uri complexe și responsive.

Concept cheie: Grid are două lucruri importante: grid-container (părintele) și grid-item (copiii).
Avantaje: aliniere facilă, zonare, gaps (spațieri), rearanjare ușoară la responsive.

🔎 Exemple teoretice + demo

Exemplu 1 — Container de bază

display: grid; + grid-template-columns pentru a defini coloane.

<div class="grid-basic">
  <div>1</div>
  <div>2</div>
  <div>3</div>
</div>

.grid-basic {
  display: grid;
  grid-template-columns: repeat(3, 1fr);
  gap: 1rem;
}

Exemplu 2 — Linie/Coloană denumită (named lines)

Poți denumi linii și apoi poziționa un item între ele: grid-column: start / end;

🧠 Exerciții practice — CSS Grid

1. Creează un layout de 3 coloane cu header și footer

Folosind grid-template-areas, rezervă o zonă pentru header, conținut și footer.

.page {
  display:grid;
  grid-template-columns: 200px 1fr;
  grid-template-rows: auto 1fr auto;
  grid-template-areas:
    "header header"
    "sidebar content"
    "footer footer";
}

Acesta este un exemplu simplu folosind grid-area. Ajustează la responsive.

2. Realizează un gallery cu carduri care reflow-uiesc la responsive

Folosește grid-template-columns: repeat(auto-fit, minmax(200px, 1fr)).

.gallery {
  display:grid;
  grid-template-columns: repeat(auto-fit, minmax(200px, 1fr));
  gap:1rem;
}

3. Provoacă: poziționează un element să ocupe 2 coloane și 2 rânduri

Folosește grid-column și grid-row.

.big {
  grid-column: span 2;
  grid-row: span 2;
}

💬 Întrebări de interviu — CSS Grid (cât mai multe)

1. Care sunt diferențele între Flexbox și Grid?

Grid = layout bidimensional (rânduri + coloane). Flexbox = unidimensional (rând sau coloană). Folosește Grid pentru layout-uri complexe.

2. Ce face grid-template-areas?

Permite definirea unei hărți vizuale a layout-ului prin string-uri care reprezintă zone numite.

3. Ce este fr unit?

Unitatea fr reprezintă o fracțiune din spațiul disponibil din containerul grid.

4. Cum funcționează grid-auto-flow?

Controlează plasarea automată a itemilor: row, column și dense pentru umplerea spațiilor.

5. Ce probleme rezolvă minmax()?

minmax(min, max) permite definirea limitelor pentru coloane/ rânduri fluide.

6. Când folosești auto-fit vs auto-fill?

auto-fit strechează grid-ul pentru a umple spațiul; auto-fill păstrează track-urile goale. Practic diferențe subtile la comportamentul colanelor.

7. Cum poziționezi elemente pe axele rând și coloană (properties)

Folosești grid-column, grid-row, justify-self, align-self pentru itemi, și justify-items, align-items pentru container.

8. Ce probleme poate crea folosirea excesivă a grid-area?

Poate face CSS-ul greu de întreținut dacă sunt folosite multe nume inconsistente; recomand structuri clare și comentarii.

9. Care este suportul browserelor pentru Grid?

Astăzi majoritatea browserelor moderne suportă CSS Grid. Verifică caniuse când ai nevoie de compatibilitate pentru versiuni vechi.

10. Cum testezi performanța layout-ului?

Testare responsive, audit Lighthouse, verificare reflow-uri excesive pe animații și evitarea recalculărilor costisitoare.

🛠️ Mini-proiect (10 taskuri) — "Dashboard Grid"

Obiectiv: construiește un mic dashboard responsive folosind CSS Grid. Fiecare task învață un concept.

  1. 🔧 Setup: creează structura HTML de bază (header, sidebar, content, footer).
  2. 📐 Grid principal: definește containerul principal cu grid-template-areas.
  3. 🧩 Sidebar: fixează dimensiunea sidebar-ului în grid (ex: 220px).
  4. 📱 Responsive: la max-width: 768px transformă layout în 1 coloană.
  5. 🖼️ Cards: creează un grid intern pentru carduri cu auto-fit/minmax.
  6. ⭐ Item mare: poziționează un widget mare care ocupă 2 coloane pe desktop.
  7. 🔁 Auto-flow: experimentează cu grid-auto-flow: dense pentru galerie.
  8. 🔒 Sticky header: folosește position: sticky pentru header sau sidebar.
  9. 🎛️ Controls: adaugă butoane care filtrează cardurile (JS minimal).
  10. 📂 Final: documentează pașii și commit-urile Git (vezi GIT_NOTES.md).

Fiecare task poate fi bifat/local testat. În arhivă găsești codul de start.
