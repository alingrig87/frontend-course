🧠 Cele mai bune practici HTML & CSS

Un ghid complet despre scrierea codului clar, semantic și performant pentru proiecte moderne.

📘 Partea Teoretică

1️⃣ Folosește HTML semantic

Tagurile precum <header>, <main> sau <article> cresc lizibilitatea și SEO-ul.

2️⃣ Clase descriptive

Evită nume generice precum .red. Preferă .error-text sau .btn-primary.

3️⃣ CSS Modular

Grupează regulile pe componente și evită duplicarea codului.

4️⃣ Mobile-First

Construiește layoutul întâi pentru ecrane mici și extinde cu media queries.

💡 Sfat: Verifică-ți codul periodic cu W3C Validator pentru a evita erorile ascunse.
⚠️ Atenție: Nu combina prea multe frameworkuri CSS – poate duce la conflicte de stiluri.

🧩 Exerciții Practice

Exercițiu 1: Creează un header semantic

Include un logo și un meniu navigațional.

<header>
  <h1>Logo</h1>
  <nav>
    <ul>
      <li><a href="#">Acasă</a></li>
      <li><a href="#">Despre</a></li>
    </ul>
  </nav>
</header>

Exercițiu 2: Stilizează un buton cu hover

Aplică tranziție lină la schimbarea culorii.

.btn {
  background: #4b3f72;
  color: white;
  border: none;
  padding: 10px 16px;
  border-radius: 6px;
  transition: background 0.3s ease;
}
.btn:hover {
  background: #7b1fa2;
}

💬 Întrebări de Interviu (HTML & CSS)

1. Care este diferența dintre tagurile <section> și <div>?

👉 <section> are valoare semantică (o secțiune logică de conținut), în timp ce <div> este doar un container generic.

2. Ce face proprietatea z-index?

👉 Controlează ordinea de afișare pe axa Z; valori mai mari apar deasupra celor mai mici.

3. Ce diferență este între em și rem?

👉 em depinde de font-size-ul părintelui, rem de font-size-ul rădăcinii (html).

🚀 Mini-Proiect: „HTML & CSS Playground”

Urmărește pașii pentru a crea un site curat, responsive și semantic.

  1. 🔧 git init – Inițializează un repository
  2. 📄 Creează fișierele index.html și style.css
  3. 🏗️ Adaugă structură semantică completă (header, main, footer)
  4. 🎨 Adaugă culori și fonturi coerente
  5. 📱 Aplică principiul Mobile First
  6. 🧩 Creează componente reutilizabile (ex: carduri)
  7. 🔍 Adaugă atribute ALT la imagini
  8. 🧭 Folosește linkuri descriptive
  9. 🧹 Rulează npx stylelint "**/*.css"
  10. 🚀 Publică pe GitHub Pages
