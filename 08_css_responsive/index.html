🔎 Partea teoretică — CSS Responsive

📱 1. Mobile-first vs Desktop-first

Mobile-first înseamnă scrierea stilurilor de bază pentru ecrane mici și adăugarea de reguli la breakpoints mai mari cu @media (min-width: ...). Desktop-first este invers (folosește max-width).

Recomandare: folosește mobile-first pentru performanță și claritate.

<!-- Mobile-first: reguli de bază pentru mobile, apoi extinderi -->
.container { padding: 12px; }
@media (min-width: 768px) {
  .container { padding: 24px; }
}

<!-- Desktop-first: reguli pentru desktop, apoi scăderi -->
.container { padding: 24px; }
@media (max-width: 767px) {
  .container { padding: 12px; }
}

🌐 2. Viewport meta & unități fluide

Include <meta name="viewport" content="width=device-width, initial-scale=1.0">. Folosește %, vw, vh, rem, em și clamp() pentru dimensiuni fluide.

h1 { font-size: clamp(1.4rem, 3.5vw, 2.2rem); }

🧭 3. Flexbox & Grid

Flexbox este excelent pentru aliniere într-o singură dimensiune, Grid pentru layout 2D. Combină-le pentru soluții robuste.

.gallery { display:flex; flex-wrap:wrap; gap:12px; }
.card { flex: 1 1 100%; }
@media (min-width:768px) {
  .card { flex: 1 1 calc(33.333% - 12px); }
}

/* sau cu Grid */
.grid { display:grid; grid-template-columns: repeat(auto-fit, minmax(220px,1fr)); gap:12px; }

🧩 4. Breakpoints practice

Breakpoints comune: 320px (mobile), 768px (tablet), 1024px (desktop). Alege breakpoints pe baza conținutului (content-first).

@media (max-width: 767px) { /* mobile */ }
@media (min-width: 768px) and (max-width: 1023px) { /* tablet */ }
@media (min-width: 1024px) { /* desktop */ }

🖼️ 5. Imagini responsive

Folosește elementele <picture>, srcset și sizes pentru a livra imagini potrivite fiecărui ecran.

<picture>
  <source media="(min-width:1024px)" srcset="img/large.jpg">
  <source media="(min-width:768px)" srcset="img/medium.jpg">
  <img src="img/small.jpg" alt="Exemplu">
</picture>

🧠 Exerciții practice (fiecare are soluția ascunsă)

Exercițiul A — Layout responsive cu Flexbox

Crează o galerie de carduri care aranjează 1 coloană pe mobile și 3 coloane pe desktop.

.gallery { display:flex; flex-wrap:wrap; gap:12px; }
.card-item { flex: 1 1 100%; }
@media (min-width: 768px) {
  .card-item { flex: 1 1 calc(33.333% - 12px); }
}

Exercițiul B — Buton cu efect hover și tranziție

Stilizează un buton astfel încât să aibă o tranziție la hover.

.btn { transition: transform 0.18s ease, box-shadow 0.18s ease; }
.btn:hover { transform: translateY(-3px); box-shadow: 0 6px 18px rgba(0,0,0,0.12); }

Exercițiul C — Utilizare clamp() pentru titluri

Setează un font-size fluid pentru h1 folosind clamp().

h1 { font-size: clamp(1.4rem, 3.5vw, 2.2rem); }

Exercițiul D — Picture + srcset

Crează un <picture> care servește o imagine mică pe mobile și una mare pe desktop.

<picture>
  <source media="(min-width:1024px)" srcset="hero-large.jpg">
  <img src="hero-small.jpg" alt="Hero">
</picture>

💬 Întrebări de interviu — CSS Responsive

1. Ce este responsive design?

👉 Responsive design înseamnă proiectarea interfețelor astfel încât să funcționeze pe o varietate de dimensiuni de ecran și dispozitive.

2. Ce face meta viewport și de ce e importantă?

👉 Tagul meta viewport controlează scala paginii pe mobile; fără el, browserele mobile afișează pagina ca pe desktop, redimensionată.

3. Cum aleg breakpoint-urile?

👉 Alege breakpoints bazat pe conținut (content-first): când designul "se rupe" vizual, adaugă o regulă media.

4. Ce este clamp()?

👉 clamp(min, val, max) definește o valoare fluidă cu limite minime și maxime — util pentru font-size sau spacing.

🧩 Mini-proiect: "Mic site responsive de portofoliu" — pași detaliați

Următorii pași sunt gândiți ca un ghid pas cu pas. După fiecare pas găsești sugestii de cod și comenzi Git.

  1. 1. Inițializează repository Git și structura proiectului

    Crează folderul proiect, rulează git init, adaugă index.html, style.css, app.js. Commit inițial:

    git init
git add .
git commit -m "Init: portfolio responsive"
  2. 2. Meta viewport și setări de bază

    Adaugă în <head>:

    <meta name="viewport" content="width=device-width, initial-scale=1.0">

    Setează baza font-size în CSS (ex: html { font-size: 16px; }) și folosește rem pentru scalare coerentă.

  3. 3. Header responsive cu hamburger

    Folosește display:flex pentru header. Adaugă butonul hamburger (vezi exemplul din pagină). JS: toggling hidden și aria-expanded.

    <button class="burger" aria-expanded="false">☰</button>
document.querySelector('.burger').addEventListener('click', () => {
  const nav = document.getElementById('mobile-nav');
  nav.classList.toggle('hidden');
});
  4. 4. Hero fluid

    Setează titlu și padding fluid:

    .hero { padding: clamp(1rem, 5vw, 3rem); }
.hero h1 { font-size: clamp(1.6rem, 4.5vw, 3rem); }
  5. 5. Galerie / secțiune proiecte

    Folosește Grid cu auto-fit sau Flexbox cu calc(). Exemplu Grid:

    .projects { display:grid; grid-template-columns: repeat(auto-fit, minmax(260px,1fr)); gap:16px; }
  6. 6. Formular de contact

    Stilează input-urile full-width și asigură spacing pe mobile:

    form input, form textarea { width:100%; padding:0.6rem; border-radius:8px; }
  7. 7. Imagini responsive (picture + srcset)

    Exemplu:

    <picture>
  <source media="(min-width:1024px)" srcset="img/project-large.jpg">
  <source media="(min-width:768px)" srcset="img/project-medium.jpg">
  <img src="img/project-small.jpg" alt="Proiect">
</picture>
  8. 8. Footer compact / sticky pe mobile

    Footer-ul poate rămâne vizibil sau compact pe mobile; folosește position:sticky doar dacă designul permite.

  9. 9. Optimizare CSS

    Folosește variabile CSS, scrie mobile-first, evită selectori foarte specifici și minimizează CSS inutil.

    :root { --primary: #4b3f72; --accent:#7b1fa2; }
  10. 10. Testare & ajustare breakpoints

    Testează la 320px, 375px, 768px, 1024px, 1440px. Ajustează paddings, font-sizes și layout.

Start code / snippet util

<!-- hamburger toggle accesibil -->
const burger = document.querySelector('.burger');
const mobileNav = document.getElementById('mobile-nav');
burger.addEventListener('click', () => {
  const open = !mobileNav.classList.toggle('hidden');
  burger.setAttribute('aria-expanded', open);
  mobileNav.setAttribute('aria-hidden', !open);
});
