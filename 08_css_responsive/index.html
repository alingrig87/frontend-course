🔎 Partea teoretică — CSS Responsive

📱 Mobile-first vs Desktop-first

Mobile-first: stilurile de bază se scriu pentru mobil. La ecrane mai mari se folosesc media queries cu min-width.
Desktop-first: stilurile de bază sunt pentru desktop. La ecrane mai mici se folosesc media queries cu max-width. 

/* Mobile-first */
.container {
  padding: 12px;
  font-size: 1rem;
}
@media (min-width: 768px) {
  .container {
    padding: 24px;
    font-size: 1.2rem;
  }
}

/* Desktop-first */
.container {
  padding: 24px;
  font-size: 1.2rem;
}
@media (max-width: 767px) {
  .container {
    padding: 12px;
    font-size: 1rem;
  }
}

🌐 Viewport & unități fluide

Include în <head>:

<meta name="viewport" content="width=device-width, initial-scale=1.0">

Unități relative și fluide recomandate:

h1 {
  font-size: clamp(1.4rem, 3vw, 2rem);
}
.section {
  width: 90%;
  max-width: 1200px;
  margin: auto;
}

🧭 Flexbox & Grid

Flexbox: pentru aliniere pe un singur rând sau coloană.
Grid: pentru layout 2D (rânduri + coloane). 

/* Flexbox */
.gallery {
  display: flex;
  flex-wrap: wrap;
  gap: 12px;
}
.card {
  flex: 1 1 100%;
}
@media (min-width: 768px) {
  .card {
    flex: 1 1 calc(33.333% - 12px);
  }
}

/* Grid */
.grid {
  display: grid;
  grid-template-columns: repeat(auto-fit, minmax(220px, 1fr));
  gap: 12px;
}

🧩 Breakpoints practice

Breakpoints recomandate: 320px (mobil), 768px (tablet), 1024px (desktop).
Alege breakpoints în funcție de conținut (content-first), nu doar dispozitiv. 

@media (max-width: 767px) { /* mobil */ }
@media (min-width: 768px) and (max-width: 1023px) { /* tablet */ }
@media (min-width: 1024px) { /* desktop */ }

🖼️ Imagini responsive

Folosește <picture>, srcset și sizes pentru a livra imagini optimizate. 

<picture>
  <source media="(min-width:1024px)" srcset="img/large.jpg">
  <source media="(min-width:768px)" srcset="img/medium.jpg">
  <img src="img/small.jpg" alt="Exemplu">
</picture>

📏 Unități și imagini scalabile

Folosește max-width și height auto pentru imagini: 

img {
  max-width: 100%;
  height: auto;
}
.container {
  width: 90%;
  max-width: 1200px;
  margin: auto;
}

🚀 Principiul "Progressive Enhancement"

Începe cu un layout simplu pe mobil și adaugă funcționalități / stiluri pentru ecrane mai mari.

🧠 Exerciții practice

Exercițiul 1 — Layout responsive cu Flexbox

Creează o galerie de carduri care aranjează 1 coloană pe mobil și 3 coloane pe desktop.

.gallery { display:flex; flex-wrap:wrap; gap:12px; }
.card-item { flex: 1 1 100%; }
@media (min-width: 768px) {
  .card-item { flex: 1 1 calc(33.333% - 12px); }
}

Exercițiul 2 — Buton cu efect hover și tranziție

Stilizează un buton astfel încât să aibă o tranziție la hover, cu ridicare și shadow subtil.

.btn { transition: transform 0.18s ease, box-shadow 0.18s ease; }
.btn:hover { transform: translateY(-3px); box-shadow: 0 6px 18px rgba(0,0,0,0.12); }

Exercițiul 3 — Titlu fluid cu clamp()

Setează un font-size fluid pentru h1 folosind clamp() pentru a se adapta la diferite lățimi de ecran.

h1 { font-size: clamp(1.4rem, 3.5vw, 2.2rem); }

Exercițiul 4 — Imagini responsive cu <picture>

Crează un <picture> care servește o imagine mică pe mobil și una mare pe desktop.

<picture>
  <source media="(min-width:1024px)" srcset="hero-large.jpg">
  <img src="hero-small.jpg" alt="Hero">
</picture>

❓ Întrebări de interviu

1. Ce înseamnă mobile-first?

Scrii CSS pentru mobil și apoi extinzi pentru desktop.

2. Ce este o media query?

Regulă CSS care aplică stiluri doar dacă o condiție e îndeplinită.

3. Cum alegem breakpoints?

Alegem breakpoints în funcție de conținut și dispozitive uzuale.

4. Care sunt unitățile relative utile în responsive design?

% (procente), em, rem, vw, vh, clamp() pentru dimensiuni fluide.

5. Exemplu practic: transformă un layout fix într-un layout fluid.

Înlocuiești width fix cu %, folosești max-width și media queries.

6. Ce capcane întâlnim la fixed width?

Layout-ul nu se adaptează pe ecrane mici, textul și imaginile pot depăși containerul.

7. Ce este content-first breakpoint?

Alegi breakpoint-urile în funcție de conținut, nu doar după dimensiunea dispozitivului.

8. Cum testezi responsive pe mobil și desktop?

Folosind DevTools în browser (mode device), resize manual sau dispozitive reale.

9. De ce folosim clamp()?

Pentru a seta dimensiuni fluide cu limite minime și maxime.

10. Care e diferența între max-width și width fix?

width fix = constant; max-width = elementul se poate micșora, dar nu depășește valoarea maximă.

🧩 Mini-proiect: "Mic site responsive de portofoliu"

1. Inițializează repository Git și structura proiectului

Cerință: Creează folderul proiect, inițializează Git și adaugă fișierele index.html, style.css, app.js.

mkdir portfolio
cd portfolio
git init
touch index.html style.css app.js
git add .
git commit -m "Init: portfolio responsive"

2. Meta viewport și setări de bază

Cerință: Fă site-ul responsive și setează font-size de bază coerent.

<meta name="viewport" content="width=device-width, initial-scale=1.0">
html { font-size:16px; }

3. Header responsive cu hamburger

Cerință: Creează header cu logo, nav și buton hamburger pentru mobil.

<header class="main-header">
  <div class="logo">🌐 Portofoliu</div>
  <button class="burger">☰</button>
  <nav class="nav">
    <a href="#">Acasă</a>
    <a href="#proiecte">Proiecte</a>
    <a href="#contact">Contact</a>
  </nav>
</header>

4. Hero fluid

Cerință: Creează secțiune hero care se scalează la diferite lățimi de ecran.

<section class="hero">
  <h1>Bine ai venit pe portofoliul meu</h1>
  <p>Proiecte responsive și moderne</p>
</section>

5. Galerie / secțiune proiecte

Cerință: Creează galerie de carduri responsive cu Flexbox sau Grid.

<section class="projects">
  <div class="gallery">
    <div class="card-item">Proiect 1</div>
    <div class="card-item">Proiect 2</div>
    <div class="card-item">Proiect 3</div>
  </div>
</section>

6. Formular de contact

Cerință: Adaugă formular cu nume, email, mesaj și buton submit.

<section id="contact">
  <form>
    <input type="text" placeholder="Nume" required>
    <input type="email" placeholder="Email" required>
    <textarea placeholder="Mesaj"></textarea>
    <button type="submit">Trimite</button>
  </form>
</section>

7. Imagini responsive

Cerință: Folosește <picture> și srcset pentru a servi imagini potrivite ecranelor.

<picture>
  <source media="(min-width:1024px)" srcset="img-large.jpg">
  <img src="img-small.jpg" alt="Imagine proiect">
</picture>

8. Footer compact / sticky

Cerință: Creează footer simplu, vizibil pe toate paginile.

<footer class="footer">
  <p>© 2025 Portofoliu</p>
</footer>

9. Optimizare CSS

Cerință: Minimizează repetarea și folosește variabile/utile pentru culori și spacing.

:root {
  --primary-color: #4b3f72;
  --secondary-color: #7b1fa2;
  --spacing: 1rem;
}
body { font-family: sans-serif; margin:0; padding:0; }

10. Testare & ajustare breakpoints

Cerință: Testează site-ul pe mobil, tablet și desktop și ajustează media queries.

@media (min-width:768px) { /* ajustări tablet */ }
@media (min-width:1024px) { /* ajustări desktop */ }
