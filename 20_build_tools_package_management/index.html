Build Tools & Package Management

În această lecție vei învăța despre ecosistemul modern JavaScript: cum gestionezi dependințele, cum folosești Webpack și Vite, cum transpilezi codul cu Babel și cum automatizezi totul pentru producție.

1. Ce sunt tool-urile de build

Tool-urile de build sunt aplicații care automatizează sarcini repetitive din procesul de dezvoltare frontend: compilare, optimizare, testare și pregătirea aplicației pentru producție.

  • ✅ Reduc dimensiunea codului (minificare)
  • ✅ Automatizează compilarea fișierelor SCSS/TypeScript
  • ✅ Optimizează imagini și cod JS/CSS
  • ✅ Rulează teste și verificări de calitate automată
# Exemple populare
- Webpack
- Vite
- Parcel
- Gulp
- Rollup
- esbuild

2. Node.js & npm recap

Node.js permite rularea JavaScript în afara browserului. npm (Node Package Manager) este sistemul de gestionare al pachetelor din ecosistemul JS. 

# Verifică versiunile instalate
node -v
npm -v

# Creează un fișier package.json automat
npm init -y

Fișierul package.json conține informații despre proiect (nume, versiune, autor) și lista de dependințe instalate.

3. Instalarea dependințelor

Pentru a adăuga o bibliotecă, folosești comanda:

# Instalare normală
npm install react

# Instalare doar pentru dezvoltare (ex: tool-uri de build)
npm install webpack --save-dev

Dependințele din secțiunea devDependencies nu sunt incluse în buildul final de producție.

4. Scripturi npm

Poți defini scripturi personalizate în fișierul package.json pentru a automatiza comenzi: 

"scripts": {
  "start": "webpack serve",
  "build": "webpack --mode production",
  "lint": "eslint src/",
  "format": "prettier --write ."
}

Rulezi scriptul astfel:

npm run build

5. Minificare & Bundling

Minificarea reduce dimensiunea fișierelor prin eliminarea spațiilor, comentariilor și denumirilor lungi. Bundling combină mai multe fișiere într-unul singur, pentru a îmbunătăți performanța. 

# Exemple
- Terser → minifică JS
- CSSNano → optimizează CSS
- Webpack → face ambele + multe altele

6. Webpack - Introducere

Webpack este unul dintre cele mai populare tool-uri pentru bundling și managementul dependințelor JS. 

# Instalare
npm install webpack webpack-cli --save-dev

Exemplu de fișier de configurare:

// webpack.config.js
module.exports = {
  entry: './src/index.js',
  output: {
    filename: 'bundle.js',
    path: __dirname + '/dist'
  },
  mode: 'development'
};

7. Module Bundling

// src/utils.js
export function salut(nume) {
  console.log(`Salut, ${nume}!`);
}

// src/index.js
import { salut } from './utils.js';
salut('developer');

Webpack combină aceste fișiere într-un singur bundle.js pentru browser.

8. Babel - Transpiling JS

Babel transformă codul modern (ES6+) în cod compatibil cu browserele mai vechi. 

# Instalare Babel
npm install --save-dev @babel/core @babel/preset-env babel-loader
// webpack.config.js
module: {
  rules: [
    {
      test: /\.js$/,
      exclude: /node_modules/,
      use: {
        loader: 'babel-loader',
        options: { presets: ['@babel/preset-env'] }
      }
    }
  ]
}

9. ESLint & Prettier

ESLint ajută la detectarea problemelor din cod, iar Prettier îl formatează automat. 

# Instalare
npm install --save-dev eslint prettier

# Inițializare ESLint
npx eslint --init

10. Vite & Tool-uri moderne

Vite este un bundler ultrarapid bazat pe ES Modules și esbuild. 

# Creare proiect Vite
npm create vite@latest my-app
cd my-app
npm install
npm run dev

11. Task Runners (Gulp)

Gulp automatizează sarcini precum compilarea SCSS, optimizarea imaginilor sau reîncărcarea automată. 

# Instalare
npm install --save-dev gulp

# gulpfile.js
const { src, dest } = require('gulp');
function copyFiles() {
  return src('src/*.html').pipe(dest('dist'));
}
exports.default = copyFiles;

12. Workshop: proiect npm + Webpack

  1. Inițializează proiectul: npm init -y
  2. Instalează Webpack și Babel
  3. Configurează webpack.config.js
  4. Adaugă scripturi npm (build, start)
  5. Rulează: npm run build

13. CI/CD & Automatizare

Tool-urile de build se integrează ușor în procese CI/CD pentru a automatiza testarea și deploy-ul. 

# Exemplu simplu GitHub Action
name: Build Frontend
on: [push]

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - run: npm install
      - run: npm run build
