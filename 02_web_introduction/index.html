💡 Introducere în Web

Acest curs explică fundamentele Internetului, World Wide Web-ului și ale arhitecturii web moderne.

📚 Partea teoretică

🌍 Internet

Internetul este o rețea globală de calculatoare care trimit pachete mici de date digitale între ele — un „sistem poștal foarte rapid”.

  • A apărut din ARPAnet în anii 1960.
  • Permite trimiterea de pagini web, emailuri, fișiere media etc.
  • Folosește suita de protocoale TCP/IP pentru comunicare.

📡 Suita de protocoale TCP/IP

Este o suită cu arhitectură pe straturi, fiecare strat având roluri și protocoale specifice:

  • Application Layer: HTTP, HTTPS, FTP, SMTP.
  • Transport Layer: TCP (fiabil), UDP (rapid dar nefiabil).
  • Network Layer: IP – adrese și rutare.
  • Data Link Layer: ARP, PPP – comunicarea hardware-rețea.

🔢 IP Addresses

Identifică fiecare calculator din rețea.

  • IPv4: format din 4 numere separate prin puncte, ex: 192.168.56.17.
  • IPv6: format din 8 grupuri hex separate prin „:”, ex: 2a04:2413:8100:8080:d4d2:c098:514d:e7b2.

🕸️ World Wide Web

WWW este un sistem de documente legate între ele prin hyperlinkuri, care permite navigarea între pagini.

Se bazează pe 3 tehnologii:

  • HTML – limbajul documentelor web.
  • URI / URL – identificarea resurselor.
  • HTTP – protocolul de transfer.

🔗 URL

Un URL (Uniform Resource Locator) indică locația unei resurse web și cum se obține aceasta.

https://developer.mozilla.org/en-US/docs/Web/HTTP/Methods/GET?source=sidebar#Syntax
  • Schemă: https
  • Domeniu: developer.mozilla.org
  • Cale: /en-US/docs/Web/HTTP/Methods/GET
  • Query: source=sidebar
  • Fragment: Syntax

💻 Arhitectura Client-Server

Modelul în care un client trimite o cerere către un server, iar serverul trimite un răspuns înapoi.

  • Exemple: Browser → Web server, Aplicație mobilă → Server API.
  • Poate fi single process (standalone) sau multi-layer.

🏗️ Arhitectura pe 3 straturi

  • Presentation: Interfața cu utilizatorul (UI/GUI).
  • Business: Logica aplicației.
  • Data: Persistența datelor (baze de date).

🌐 DNS

DNS (Domain Name System) este „agenda telefonică” a internetului: traduce nume de domenii în adrese IP.

📨 HTTP

HTTP (HyperText Transfer Protocol) este protocolul de comunicare între client și server pentru resurse web.

  • Model Request → Response
  • Text-based
  • Stateless (fiecare cerere e independentă)

🔁 Metode HTTP

  • GET: Citire (Read)
  • POST: Creare (Create)
  • PUT: Actualizare (Update)
  • DELETE: Ștergere (Delete)

📦 Răspunsuri HTTP

  • 1xx – Informativ
  • 2xx – Succes (ex: 200 OK, 201 Created)
  • 3xx – Redirectare (ex: 302 Found)
  • 4xx – Eroare client (ex: 404 Not Found)
  • 5xx – Eroare server (ex: 500 Internal Server Error)

🔒 HTTPS

HTTPS = HTTP + SSL/TLS (criptare pentru securitate). Protejează confidențialitatea și integritatea datelor transmise.

🧭 Web Browsers

Un browser este o aplicație client care afișează conținutul de pe web.

  • Chrome
  • Firefox
  • Edge
  • Safari
  • Opera

🧰 Developer Tools

Instrumente integrate în browsere pentru dezvoltatori:

  • Inspectare DOM și CSS
  • Console pentru debugging JS
  • Analiză Network și Performance

🧠 Exerciții

1️⃣ Care sunt cele 4 straturi ale suitei TCP/IP?

Încearcă să le enumeri...

Application, Transport, Network, Data Link.

2️⃣ Descompune un URL în părțile sale.

Schemă, domeniu, cale, query, fragment.

3️⃣ Care este diferența între HTTP și HTTPS?

HTTPS criptează datele cu SSL/TLS, spre deosebire de HTTP care transmite text clar.
